# 智企连·AI-Ready 项目方案

**基于 SmartAdmin 框架，借鉴 Odoo 设计思想，打造完全自主、适合中国用户的 AI-Ready 企业管理系统**

---

## 1. 项目背景与目标

### 1.1 背景
- **市场需求**：中小企业对一体化 ERP+CRM 系统需求旺盛，但传统 SaaS 产品存在数据安全、定制成本高、按年付费等问题；同时，AI 技术的成熟使得"用自然语言指挥业务"成为可能，企业需要能被 AI Agent 调用的业务系统。
- **技术趋势**：国内头部 ERP 厂商（用友、金蝶等）正在加速 AI 融合，而开源社区缺乏一款符合中国用户习惯、技术栈主流、自主可控的 AI-Ready ERP。
- **前期讨论**：经过多轮技术选型与合规分析，团队决定从基于 Odoo 二次开发转向**完全自主开发**，以规避开源协议风险、掌握全部知识产权、适配国内技术生态。

### 1.2 目标
打造一款 **开源与商业相结合** 的企业智能管理系统，具备以下核心特性：
- **AI-Ready**：预留标准化的 Agent 调用接口，任何 AI Agent（如 OpenClaw、Dify、LangChain、自研 Agent）均可调用系统能力。
- **模块化**：借鉴 Odoo 的设计哲学，业务模块可插拔、可独立扩展、可组合部署。
- **中国化**：内置中国会计准则、电子发票接口、银行直连、移动端多端适配（小程序、APP、H5）。
- **自主可控**：100% 自研代码，采用 **Apache 2.0** 开源协议，无任何法律风险，企业可安心商用。
- **生产级**：基于成熟的 SmartAdmin 框架，满足安全合规、高性能、高可用要求。

## 3. 技术选型

### 3.1 基础框架：SmartAdmin

**选择理由**：
- **安全合规**：国内首个满足《网络安全-三级等保》要求、内置数据脱敏、接口加解密、登录防爆破等功能。
- **双版本支持**：Java8+SpringBoot2 或 Java17+SpringBoot3，满足不同部署环境。
- **成熟代码规范**：开源六年，千余企业验证的代码规范，降低协作成本。
- **内置代码生成器**：一键生成前后端 CRUD 代码，大幅提升开发效率。
- **移动端原生支持**：基于 UniApp 的移动端模板，可直接扩展多端应用。

**技术栈明细**：
| 层次 | 技术 | 说明 |
|------|------|------|
| 后端 | Java 17 / Spring Boot 3.2.x | 长期支持版本 |
| 安全框架 | Sa-Token | 轻量级权限认证，支持 OAuth2.0 |
| ORM | Mybatis-Plus | 增强 CRUD，提供丰富条件构造器 |
| 数据库 | MySQL 8.0 / PostgreSQL | 主库 MySQL，可选 PG 支持 |
| 缓存 | Redis 7.x | 会话、数据缓存 |
| 消息队列 | RocketMQ / RabbitMQ | 异步任务、打印队列 |
| 前端 | Vue 3 + Vite 5 + Ant Design Vue | 现代化组件库 |
| 移动端 | UniApp + Vue 3 | 一套代码编译为 H5、微信小程序、APP |
| API 文档 | Knife4j | 自动生成 OpenAPI 规范 |
| 容器化 | Docker + Kubernetes | 便于交付部署 |

### 3.2 借鉴 Odoo 的设计模式

| Odoo 特性 | 借鉴实现方式 | 落地模块 |
|-----------|--------------|----------|
| **模块化架构** | Maven 多模块项目，每个业务模块独立打包，通过 SPI 或 Spring 自动装配进行集成。 | `erp-*`、`crm-*` 等模块 |
| **模型继承** | 使用 Java 类继承 + 组合模式；通过 `@TableField` 扩展自定义字段。 | 基类 `BaseEntity`，业务实体继承 |
| **视图继承/定制** | 前端组件插槽（slot）+ 混入（mixin），后端提供 JSON 配置化表单。 | Vue 组件库 + 低代码表单设计器 |
| **ORM 高级特性** | Mybatis-Plus 提供 `QueryWrapper`，自定义 `@TableField(exist=false)` 实现计算字段。 | 所有 Service 层 |
| **动态字段** | 通过 JSON 列或扩展表存储动态字段，前端动态渲染。 | `custom_field` 表 + 动态表单组件 |
| **工作流引擎** | 集成 Flowable 或 Camunda，提供图形化流程设计器。 | `core-workflow` 模块 |
| **报表引擎** | 集成 JasperReports 或自研基于 HTML+PDF 的打印模板。 | `core-report` 模块 |

## 5. 核心功能模块设计

### 5.1 ERP 模块群（独立顶层模块）

| 模块 | 核心功能 | 中国化特性 |
|------|----------|------------|
| **销售管理** | 报价、销售订单、发货、退货、客户信用 | 支持增值税发票、电子发票接口（百望/诺诺） |
| **采购管理** | 采购申请、询价、采购订单、入库、供应商对账 | 支持电子采购合同、供应商三证管理 |
| **库存管理** | 入库、出库、盘点、调拨、库存预警 | 支持批次/序列号、保质期管理、多单位换算 |
| **仓储管理** | 货位管理、PDA 拣货、波次策略、上架策略 | 支持仓库分区、货架条码打印 |
| **财务管理** | 应收应付、总账、报表、固定资产 | 内置中国会计科目表、凭证模板、金税接口、财务报表（资产负债表/利润表） |

### 5.2 CRM 模块群（独立顶层模块）

| 模块 | 核心功能 | 中国化特性 |
|------|----------|------------|
| **线索管理** | 线索录入、分配、跟进、转化 | 支持企业工商信息自动补全（天眼查 API） |
| **商机管理** | 商机阶段、预测金额、竞争分析 | 支持销售漏斗、回款计划 |
| **客户管理** | 客户档案、联系人、互动历史、客户分级 | 支持客户信用评级、黑名单 |
| **活动记录** | 电话、拜访、邮件记录 | 集成企业微信、钉钉消息 |

### 5.3 Agent 调用层（核心差异化）

- **能力注册中心**：启动时扫描 `@AgentCapability` 注解，将能力注册到数据库表 `agent_capability`。
- **统一调用协议**：使用 JSON-RPC 2.0，支持 HTTP 同步和 WebSocket 异步。
- **安全与审计**：每个 Agent 调用需要携带 API Key，记录调用日志（包括 Agent 标识、操作人、参数、结果）。
- **能力示例**（覆盖 ERP 与 CRM）：
  - `sale.order.create`：创建销售订单
  - `sale.order.query`：查询订单
  - `stock.picking.print`：打印拣货单
  - `crm.lead.create`：创建线索
  - `crm.opportunity.update`：更新商机阶段
  - `report.sales.today`：今日销售报表

**接口规范**（JSON-RPC）：
```json
{
  "jsonrpc": "2.0",
  "method": "capability.invoke",
  "params": {
    "capability_id": "sale.order.create",
    "context": {
      "session_id": "conv_12345",
      "user_id": 10086,
      "agent_id": "openclaw_001"
    },
    "arguments": {
      "customer_id": 123,
      "products": [{"id": 456, "qty": 2}]
    }
  },
  "id": 1
}
```

## 6. 前端多端规划

| 前端 | 技术方案 | 核心功能 | Agent 交互 |
|------|----------|----------|------------|
| **管理前端** | Vue3 + Ant Design Vue (SmartAdmin 自带) | 全功能后台：ERP/CRM 操作、报表、配置、审批 | 集成 AI 对话组件，可调用 Agent 能力 |

| **用户订货商城** | UniApp (H5/小程序) | 商品浏览、下单、订单跟踪、在线支付 | Agent 可代为下单（客服场景） |

## 9. 风险管理

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 团队对 SmartAdmin 不熟悉 | 开发效率低 | 初期组织培训，编写内部使用指南，安排技术骨干先行探索 |
| Odoo 设计思想落地偏差 | 架构混乱 | 每周技术评审，参考 Odoo 官方文档，提炼核心模式并适配 Java 实现 |
| Agent 调用层性能瓶颈 | AI 交互延迟 | 采用异步处理 + 缓存，关键能力支持批量调用 |
| 国产化接口（税务、银行）变化 | 合规风险 | 模块化设计，预留配置接口；与专业服务商合作，定期更新 |
| 交付延期 | 影响市场节奏 | 分阶段交付，优先核心功能，MVP 后快速迭代 |
