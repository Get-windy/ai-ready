## 一、项目概览

### 1.1 项目定位

**智企连·AI-Ready** 是基于 SmartAdmin 框架，借鉴 Odoo 设计思想，打造完全自主、适合中国用户的 AI-Ready 企业管理系统。

**核心特性**:
- **AI-Ready**: 预留标准化的 Agent 调用接口，任何 AI Agent（如 OpenClaw、Dify、LangChain、自研 Agent）均可调用系统能力
- **模块化**: 借鉴 Odoo 的设计哲学，业务模块可插拔、可独立扩展、可组合部署
- **中国化**: 内置中国会计准则、电子发票接口、银行直连、移动端多端适配（小程序、APP、H5）
- **自主可控**: 100% 自研代码，采用 Apache 2.0 开源协议，无任何法律风险，企业可安心商用
- **生产级**: 基于成熟的 SmartAdmin 框架，满足安全合规、高性能、高可用要求

### 1.2 项目架构

**技术栈**:
- **后端**: Java 17 / Spring Boot 3.2.x / Sa-Token / Mybatis-Plus
- **前端**: Vue 3 + Vite 5 + Ant Design Vue
- **移动端**: UniApp (多端)
- **数据库**: MySQL 8.0 / PostgreSQL
- **缓存**: Redis 7.x
- **消息队列**: RocketMQ / RabbitMQ
- **容器化**: Docker + Kubernetes

**项目结构**:
```
I:\AI-Ready
├── core-base                       # 基础支撑模块（用户、权限、配置、字典、通用工具）
├── core-workflow                   # 工作流模块（集成 Flowable）
├── core-report                     # 报表与打印模块
├── core-agent                      # Agent 调用层模块
├── core-api                        # API 接口定义（供前端调用）
├── core-common                     # 公共工具类
├── erp                             # ERP 模块群（独立顶层）
│   ├── erp-purchase                # 采购管理
│   ├── erp-sale                    # 销售管理
│   ├── erp-stock                   # 库存管理
│   ├── erp-warehouse               # 仓储管理（PDA拣货等）
│   ├── erp-account                 # 财务管理（中国化）
│   └── erp-production              # 生产管理（可选）
├── crm                             # CRM 模块群（独立顶层）
│   ├── crm-lead                    # 线索管理
│   ├── crm-opportunity             # 商机管理

├── smart-admin-web                 # 管理前端（Vue3 项目）
└── erp-mobile                      # 移动端 UniApp 项目（多端）
```

## 六、高可用架构设计

### 6.1 架构目标

- **可用性目标**: 99.9% (年度停机时间 < 8.76小时)
- **RTO (恢复时间目标)**: < 15分钟
- **RPO (恢复点目标)**: < 5分钟
- **性能目标**: API响应时间 < 200ms (P95)

### 6.2 架构层级

| 层级 | 组件 | 可用性 | 说明 |
|------|------|--------|------|
| 接入层 | CDN/WAF | 99.99% | DDoS防护、静态资源加速 |
| 负载均衡层 | NGINX + Keepalived | 99.99% | 双机热备、健康检查 |
| 应用层 | Spring Boot 集群 | 99.95% | 多实例部署、自动伸缩 |
| 缓存层 | Redis Cluster | 99.99% | 主从复制、哨兵模式 |
| 数据层 | PostgreSQL Cluster | 99.99% | 主从复制、自动故障转移 |

### 6.3 监控与告警

**监控系统**:
- Prometheus: 指标收集
- Grafana: 可视化监控
- AlertManager: 告警管理
- Loki: 日志收集

**告警规则**:
- 应用层: CPU > 80%、内存 > 85%、响应时间 > 500ms
- 数据库: 连接数 > 80%、慢查询 > 1s
- 缓存: 内存使用 > 90%、命中率 < 80%
