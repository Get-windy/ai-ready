## 五、性能优化建议

### 查询优化 🔴 高优先级
1. 为常用查询字段添加索引
2. 避免 SELECT *，只查询需要的字段
3. 使用 EXPLAIN 分析慢查询
4. 优化复杂 JOIN 操作

### 写入优化 🟡 中优先级
1. 使用批量插入代替单条插入
2. 合理使用事务
3. 避免大事务

### 并发优化 🟡 中优先级
1. 根据CPU核心数调整连接池大小
2. 考虑读写分离
3. 大表考虑分库分表

### 缓存优化 🟢 低优先级
1. 热点数据使用多级缓存
2. 设置合理的缓存过期时间
3. 实现缓存预热
