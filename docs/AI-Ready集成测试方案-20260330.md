# AI-Ready 集成测试方案

## 1. 概述

### 1.1 测试目标
验证AI-Ready系统各模块间的协同工作能力，确保：
- 模块间接口调用正确
- 数据流转符合业务逻辑
- 权限控制在模块间正确传递
- 异常处理机制完整

### 1.2 测试范围
| 模块 | 说明 | 接口数 | 依赖模块 |
|------|------|--------|----------|
| 用户管理 | 用户CRUD、认证、授权 | 20+ | 核心平台 |
| 角色权限 | 角色管理、权限分配 | 15+ | 核心平台 |
| ERP模块 | 销售订单、采购订单、库存管理 | 30+ | 核心平台 |
| CRM模块 | 客户、线索、商机管理 | 25+ | 核心平台 |
| Agent服务 | Agent管理、能力调用 | 10+ | 核心平台 |

### 1.3 测试环境
- 测试环境: http://localhost:8080
- 数据库: PostgreSQL (Docker)
- 缓存: Redis (Docker)

## 2. 模块分析

### 2.1 核心平台模块

#### 2.1.1 用户服务 (SysUserService)
**核心功能**:
- 用户注册/登录/登出
- 用户信息查询/更新
- 密码修改/重置

**接口依赖**:
```
AuthController (认证)
├── POST /user/login - 登录
├── POST /user/logout - 登出
└── GET /user/info - 获取用户信息

SysUserController (用户管理)
├── GET /user/page - 分页查询
├── GET /user/{id} - 查询详情
├── POST /user - 创建用户
├── PUT /user/{id} - 更新用户
└── DELETE /user/{id} - 删除用户
```

#### 2.1.2 角色权限服务 (SysRoleService/SysPermissionService)
**核心功能**:
- 角色创建/查询/更新/删除
- 权限配置
- 菜单权限分配

**接口依赖**:
```
SysRoleController
├── GET /role/page - 分页查询
├── GET /role/{id} - 查询详情
├── POST /role - 创建角色
├── PUT /role/{id} - 更新角色
├── POST /role/{id}/permissions - 分配权限
└── DELETE /role/{id} - 删除角色

SysPermissionController
├── GET /permission/page - 权限列表
└── POST /permission - 创建权限
```

## 3. 集成测试场景设计

### 3.1 用户-权限-角色集成测试

#### 场景1: 用户注册并分配角色
**测试步骤**:
1. 管理员调用创建用户API创建新用户
2. 调用角色分配API为用户分配角色
3. 验证用户能正常登录
4. 验证用户根据角色拥有对应权限

**预期结果**:
- ✅ 用户创建成功
- ✅ 角色分配成功
- ✅ 用户能使用Token登录
- ✅ 用户访问授权API返回正确权限列表

#### 场景2: 角色权限 inheritance
**测试步骤**:
1. 创建角色A（基础权限）
2. 创建角色B（继承角色A权限）
3. 为用户分配角色B
4. 验证用户同时拥有A和B的权限

**预期结果**:
- ✅ 角色B继承角色A的所有权限
- ✅ 用户拥有合并后的权限集

### 3.2 ERP-CRM集成测试

#### 场景3: 销售订单流程（CRM → ERP）
**测试步骤**:
1. CRM模块创建新客户
2. CRM模块为客户创建线索
3. CRM模块将线索转化为商机
4. ERP模块创建销售订单（关联客户）
5. ERP模块创建订单详情（关联商机）
6. 验证订单信息正确

**预期结果**:
- ✅ 客户信息同步到订单
- ✅ 商机信息关联到订单
- ✅ 订单状态正确流转

#### 场景4: 库存-销售订单联动
**测试步骤**:
1. 创建销售订单（商品A: 100件）
2. 验证库存减少100件
3. 订单发货后，库存状态更新
4. 订单退货后，库存恢复

**预期结果**:
- ✅ 订单创建时库存正确减少
- ✅ 发货后库存状态标记为"已出库"
- ✅ 退货后库存恢复

### 3.3 Agent服务集成测试

#### 场景5: Agent能力注册与调用
**测试步骤**:
1. Agent服务注册新Agent
2. 验证Agent在Agent列表中可见
3. 调用Agent能力API
4. 验证返回正确的Agent能力信息

**预期结果**:
- ✅ Agent注册成功
- ✅ Agent能力API返回正确数据
- ✅ Agent列表包含新注册的Agent

#### 场景6: JSON-RPC集成调用
**测试步骤**:
1. 通过JSON-RPC调用Agent服务
2. 验证返回格式符合JSON-RPC 2.0规范
3. 测试错误处理（method不存在）
4. 测试错误处理（参数错误）

**预期结果**:
- ✅ 正确返回JSON-RPC格式响应
- ✅ 错误返回coding字段正确
- ✅ 错误信息清晰明确

### 3.4 全链路集成测试

#### 场景7: 用户全流程（从注册到订单）
**测试步骤**:
1. 用户注册（用户服务）
2. 管理员分配角色（角色服务）
3. 用户登录获取Token（认证服务）
4. 用户创建商机（CRM服务，使用Token）
5. 将商机转化为订单（销售订单服务）
6. 验证订单创建者为该用户

**预期结果**:
- ✅ 全流程数据一致
- ✅ 每个环节权限验证正确
- ✅ 用户能访问授权范围内的数据

#### 场景8: 多角色用户权限混合
**测试步骤**:
1. 创建用户U1，分配角色A（销售）
2. 创建用户U2，分配角色B（采购）
3. 创建角色C，包含A和B的权限
4. 将角色C分配给U1
5. 验证U1能访问销售和采购相关API

**预期结果**:
- ✅ U1拥有角色C的完整权限
- ✅ U2只能访问采购API
- ✅ 权限隔离正确
