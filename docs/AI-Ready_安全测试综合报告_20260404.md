## 四、修复建议

### 4.1 高优先级修复 🔴

**1. 增强SQL注入检测**

- **问题**: URL编码Payload未被检测
- **影响**: 可能绕过SQL注入防护
- **修复方案**: 
  ```java
  // 在SQL注入过滤器中添加URL解码检测
  String decodedParam = URLDecoder.decode(param, "UTF-8");
  if (containsSqlInjection(decodedParam)) {
      throw new SecurityException("SQL注入检测");
  }
  ```
- **预计工时**: 2小时

**2. 完善XSS防护**

- **问题**: 事件处理器属性转义不完整
- **影响**: 可能存在XSS攻击风险
- **修复方案**:
  ```java
  // 完善XSS清理，移除所有事件处理器属性
  String cleanHtml = Jsoup.clean(input, Whitelist.relaxed());
  // 额外移除on*事件属性
  cleanHtml = cleanHtml.replaceAll("on\\w+\\s*=", "");
  ```
- **预计工时**: 2小时

### 4.2 中优先级改进 🟡

**1. 实现API频率限制**

- **建议**: 配置基于IP和用户的频率限制
- **推荐配置**:
  | API类型 | 限制 |
  |---------|------|
  | 普通API | 100 req/min |
  | 敏感API | 20 req/min |
  | 登录API | 5 req/min |
- **预计工时**: 4小时

**2. 完善Token管理**

- **建议**: 实现Token黑名单机制
- **功能**: 
  - 登出时Token立即可撤销
  - 异地登录检测
  - Token使用审计
- **预计工时**: 4小时

### 4.3 低优先级优化 🟢

**1. 增强审计日志**

- **建议**: 添加更详细的操作审计
- **内容**: 
  - 敏感数据访问记录
  - 批量导出记录
  - 异常操作告警

**2. 实现字段级加密**

- **建议**: 对敏感字段实施AES-256加密
- **字段**: 身份证号、银行卡号等
