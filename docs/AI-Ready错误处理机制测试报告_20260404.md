## 4. 错误处理架构

### 4.1 错误处理流程

```
┌─────────────┐
│   API请求   │
└──────┬──────┘
       │
       ▼
┌─────────────┐     ┌─────────────┐
│  参数验证    │────>│  验证失败    │
└──────┬──────┘     └──────┬──────┘
       │                   │
       ▼                   ▼
┌─────────────┐     ┌─────────────┐
│  业务处理    │────>│  业务错误    │
└──────┬──────┘     └──────┬──────┘
       │                   │
       ▼                   ▼
┌─────────────┐     ┌─────────────┐
│   成功响应   │     │ 错误响应构建 │
└─────────────┘     └──────┬──────┘
                           │
                           ▼
                    ┌─────────────┐
                    │ 统一错误格式 │
                    │ code/msg/ts │
                    └─────────────┘
```

### 4.2 标准错误响应字段

| 字段 | 类型 | 必需 | 描述 |
|------|------|------|------|
| code | int | ✅ | 业务错误码 |

| timestamp | string | ✅ | ISO时间戳 |
| success | boolean | ✅ | 固定false |
